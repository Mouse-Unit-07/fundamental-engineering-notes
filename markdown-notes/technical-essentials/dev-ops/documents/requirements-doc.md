## Sample Contents

- ...If there was a universal format for requirements docs, we would've encountered it in the _DevOps Handbook_ and other Agile books
- A sample from _Embedded Systems: Introduction to ARM Cortex-M Microcontrollers_ outlines:
- Overview
  - Objectives
    - Why the project exists / purpose
  - Process
    - How the project will be engineered
  - Roles and responsibilities
    - Roles of those involved, target clients
  - Interactions w/ existing systems
    - How the new system will fit in
  - Terminology
    - Technical/non-universal terms used in the document
  - Security
    - How the new system will be managed
- Function Description
  - Functionality
    - What the new system will do
  - Scope
    - Phases of development and what they'll deliver
  - Prototypes
    - How intermediate progress will be demonstrated
  - Performance
    - Measures of performance and how they'll be determined
  - Usability
    - How the new system will be interfaced w/
    - Include quantitative descriptions if applicable
